# Artifact Hygiene Policy

Maintainer note: internal implementation/release artifact; not a public readiness claim.


Last updated: 2026-04-01

This policy defines which generated artifacts must be ignored, cleaned, archived, or deleted before release verification.

## Generated Artifact Classes

| Class | Paths | Policy |
| --- | --- | --- |
| Node dependencies/build outputs | `node_modules/`, `dist/`, `apps/desktop/dist/` | Ignore in VCS. Rebuild per environment. |
| Sidecar packaging scratch outputs | `build/`, `vivid-inference-sidecar.spec` | Ignore in VCS. Generated by sidecar bundling; safe to delete between runs. |
| Python caches | `__pycache__/`, `*.pyc`, `services/inference/.pytest_cache/`, `services/inference/.mypy_cache/` | Ignore in VCS. Safe to delete at any time. |
| Python package metadata artifacts | `services/inference/*.egg-info/` (including `services/inference/vivid_inference.egg-info/`) | Ignore in VCS. Regenerated by editable installs/build tooling. |
| Tauri/Rust build outputs | `apps/desktop/src-tauri/target/` | Ignore in VCS. Rebuild in CI/release jobs. |
| Sidecar staged binaries | `apps/desktop/src-tauri/binaries/*` (except README) | Keep out of VCS. Stage during build pipeline only. |
| E2E runtime app data | `.e2e-data/` | Ignore in VCS. Delete between test runs if state pollution is suspected. |
| Playwright outputs | `apps/desktop/test-results/`, `apps/desktop/playwright-report/` | Ignore in VCS. Archive as CI artifacts only. |
| Local release/signing material | `.secrets/` | Never commit. Restricted local access only. |
| Local logs/temp | `*.log`, `Thumbs.db`, `.DS_Store` | Ignore in VCS. Delete as needed. |

## Pre-Release Cleanup Checklist

Run this before final RC tagging:

1. Confirm no generated artifacts are staged for commit.
2. Remove local runtime state used only for testing:
   - `.e2e-data/`
   - `apps/desktop/test-results/`
   - `apps/desktop/playwright-report/` (if present)
   - `services/inference/*.egg-info/` (if present)
   - `build/` and `vivid-inference-sidecar.spec` (if present)
3. Verify no secret material is tracked:
   - `.secrets/`
4. Re-run critical checks from a clean build context:
   - backend tests
   - desktop build
   - Playwright smoke

Record the cleanup evidence in [`docs/RELEASE_GATES_V1.md`](./RELEASE_GATES_V1.md) gate `RG-00`.
